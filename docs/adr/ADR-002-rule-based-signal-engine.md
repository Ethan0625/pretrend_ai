# ADR-002: Rule-based Signal Engine (ML 모델 대신 상태 기반 규칙 엔진)

Markers: architecture, contract
Status: reference

## 상태
채택

## 날짜
2026-01-15 (초기 설계), 2026-02-13 (구조 확정)

## 배경

전략 신호(Long/Mid/Short 시장 상태 판단)를 생성하는 방법으로
ML 모델과 규칙 기반 엔진 중 하나를 선택해야 했다.

이 시스템은 아래 조건을 전제로 한다.

- **데이터셋 크기**: 2006~2026년 약 5,000거래일. Regime 변환(GFC, COVID, 금리인상기 등)을 포함하면
  실질적으로 동질적인 학습 데이터는 더 적다.
- **운영 환경**: 매일 단 1회 실행. 실시간 추론이 아님.
- **신뢰 요건**: 각 매매 결정의 근거를 사후에 추적하고 설명할 수 있어야 한다.
- **안정성 요건**: 모델 재학습 없이 장기 운영 가능해야 한다.

---

## 결정

**Long/Mid/Short 시장 상태 판단을 ML 모델이 아닌 규칙 기반 상태 기계(State Machine)로 구현한다.**

- Long Engine: FRED 거시 지표 delta_6m rolling z-score → EXPANSION/RECOVERY/LATE_CYCLE/SLOWDOWN/RECESSION
- Mid Engine: 가격 모멘텀 + 거시 정책 방향 + 시장 breadth 3-신호 다수결 → RISK_ON/NEUTRAL/RISK_OFF
- Short Engine: 일별 수익률 + 변동성 + 보조 신호(4~5개) 조합 → PANIC/NORMAL/RELIEF

각 엔진은 명시적 임계값과 조건으로 구성되며, 모든 판단이 입력값과 규칙으로 완전히 추적 가능하다.

---

## 근거

**1. 데이터 부족 문제**
금융 시계열에서 Regime 변환은 드물고 비선형적이다.
GFC(2008), COVID(2020), 금리인상기(2022) 같은 극단 구간은 데이터셋에 2~3회 존재한다.
이 데이터로 ML 모델을 학습하면 과거 Regime에 과적합(overfitting)될 위험이 크다.
규칙 기반 엔진은 Regime이 바뀌어도 규칙의 의미가 유지된다.

**2. 재현성(Reproducibility) 보장**
ML 모델은 학습 데이터, 시드, 하이퍼파라미터, 프레임워크 버전에 따라 출력이 달라질 수 있다.
규칙 기반 엔진은 동일 입력 → 동일 출력이 항상 보장된다.
이는 Backtest와 Paper Trading 결과 비교의 전제 조건이다.

**3. 설명 가능성(Explainability)**
매매 결정의 근거를 규제 준수, 운영 검증, 전략 연구 목적으로 추적해야 한다.
"Long=RECESSION이라 DECREASE 신호 발생"은 설명 가능하지만,
"모델이 0.73 확률로 하락을 예측"은 근거를 소급 추적하기 어렵다.

**4. 운영 안정성**
ML 모델은 성능 드리프트 모니터링, 주기적 재학습, 모델 버전 관리가 필요하다.
일 1회 실행되는 운영 시스템에서 이 overhead는 핵심 가치 대비 비용이 크다.
규칙 기반 엔진은 임계값 조정 외 별도 재학습 작업이 없다.

**5. Fail-open 설계 가능**
데이터가 없거나 계산 불가한 경우 `UNKNOWN` 상태를 반환하고 시스템이 계속 작동한다.
ML 모델은 입력 분포가 학습 범위를 벗어날 때 예측 불안정성이 커진다.

---

## 검토한 대안

| 대안 | 거부 이유 |
|---|---|
| XGBoost/RandomForest 분류 모델 | Regime 과적합 위험, 재현성 미보장, 설명 불가 |
| LSTM/Transformer 시계열 모델 | 학습 데이터 부족, 운영 재학습 부담, 블랙박스 |
| 강화학습 | 환경 정의 복잡, 샘플 효율 낮음, 실 운영 안정성 미검증 |
| 규칙 + ML 앙상블 | 복잡성 증가 대비 명확한 이점 없음, 설명 가능성 훼손 |

---

## 영향

**수용한 트레이드오프**
- 새로운 시장 패턴에 자동 적응하지 못함. 임계값 조정은 수동 연구 과정 필요.
- 비선형 복합 패턴을 규칙으로 표현하는 데 한계 존재.

**얻은 것**
- 모든 매매 결정의 완전한 추적 가능성
- 학습 없이 장기 운영 가능
- Backtest ↔ Paper Trading ↔ Mock Trading 결과 비교의 신뢰성
- 전략 연구 시 임계값 변경 효과를 Backtest로 즉시 검증 가능
- GFC/COVID/금리인상기 실증 검증 통과 (MEMORY 참조)
