# ADR-005: LLM을 해석 계층으로만 한정 (의사결정 엔진 배제)

Markers: architecture, contract
Status: reference

> 🔄 **현재 유효한 설명 계층 참고 기록** — explainability layer 원칙 (관측 only, 예측 금지). 분류: [`README.md`](../../README.md)

## 상태
채택

## 날짜
2026-02-01 (설계 원칙 확정)

---

## 배경

LLM(Large Language Model)을 이 시스템에 통합할 때 역할을 어떻게 정의할 것인지 결정해야 했다.

LLM을 활용할 수 있는 범위는 크게 세 가지였다.

1. **의사결정 참여**: LLM이 뉴스/공시/시장 상황을 분석하여 매매 신호를 직접 생성
2. **신호 보정**: 규칙 기반 신호에 LLM 판단을 가중치로 반영
3. **해석 및 보고**: 이미 결정된 전략 결과를 LLM이 설명하고 리포트를 작성

이 결정은 시스템 신뢰성, 운영 비용, 재현성 요건에 직접적인 영향을 미친다.

---

## 결정

**LLM은 해석 계층(Interpretation Layer)으로만 사용한다. 매매 신호 생성이나 의사결정에 관여하지 않는다.**

```
[데이터 레이어] → [신호 엔진 (rule-based)] → [Allocation Engine] → [실행]
                                                        ↓
                                               [LLM 해석 계층]
                                               - 리포트 작성
                                               - 전략 근거 설명
                                               - 리서치 요약
```

LLM은 전략 판단의 결과물(Action, Phase, Regime 등)을 입력받아
자연어 설명과 리포트를 생성하는 역할만 수행한다.

---

## 근거

**1. 재현성 보장 불가**
LLM 출력은 동일 입력에 대해서도 확률적으로 다른 출력을 낼 수 있다.
동일 날짜의 전략 판단을 재현해야 할 때 LLM이 의사결정에 관여하면
이를 보장할 방법이 없다.
규칙 기반 엔진은 동일 입력 → 동일 출력이 항상 보장된다.

**2. 환각(Hallucination) 리스크**
LLM은 사실과 다른 정보를 그럴듯하게 생성할 수 있다.
"CPI가 3.2%라서 긴축 가능성이 낮다"는 문장이 틀린 수치를 기반으로 할 때
이를 실시간으로 검증할 방법이 없다.
매매 신호가 환각에 기반하면 손실로 직결된다.

**3. 비용과 지연**
LLM API 호출은 비용이 발생하고 수십~수백ms의 지연이 있다.
매일 실행되는 전략 파이프라인이 LLM 가용성에 의존하면
네트워크 장애나 API 제한으로 전략 실행 자체가 실패할 수 있다.
해석 계층은 전략 실행 이후에 독립적으로 실행되므로 이 의존성이 없다.

**4. 감사 추적(Audit Trail) 유지**
금융 의사결정은 근거가 명확히 추적돼야 한다.
"Long=RECESSION이고 Mid=RISK_OFF이므로 DECREASE" 경로는 완전히 추적된다.
LLM이 개입한 결정은 어떤 정보를 어떻게 가중해 결정했는지 추적이 불가능하다.

**5. LLM의 실질적 강점 활용**
LLM이 실제로 강한 영역은 자연어 생성과 정보 요약이다.
이미 계산된 전략 결과를 사람이 이해하기 쉬운 언어로 설명하는 데 LLM은 탁월하다.
이 역할에 집중하면 LLM의 강점을 활용하면서 리스크를 격리할 수 있다.

---

## 검토한 대안

| 대안 | 거부 이유 |
|---|---|
| LLM 기반 매매 신호 생성 | 재현성 불가, 환각 리스크, 감사 추적 불가 |
| LLM + 규칙 앙상블 | 비결정성 오염, 트레이드오프 분석 복잡, 설명 불가 |
| LLM RAG 기반 뉴스 신호 | 현재 단계에서 뉴스 데이터 수집 인프라 미비, 향후 확장 포트로 보존 |

---

## 영향

**수용한 트레이드오프**
- 뉴스/공시 기반 단기 이벤트 대응 불가. 현재 구조에서 LLM은 사후 설명만 가능.
- 자연어 정보(뉴스 센티멘트 등)의 정량적 활용 미착수.

**얻은 것**
- 전략 파이프라인의 완전한 재현성 유지
- LLM 가용성이 전략 실행에 영향 없음 (해석 계층은 독립 실행)
- 모든 매매 결정의 완전한 감사 추적 가능
- LLM 비용이 리포트 생성에만 발생 (전략 실행 횟수와 무관)

**향후 확장**
- 뉴스/공시 기반 Text 신호 레이어는 Reserved Port로 설계에 명시됨
- 해당 데이터 인프라 구축 후 Mid Engine 또는 별도 레이어로 통합 가능
- LLM은 이때도 해석 계층 원칙 유지 예정 (신호 생성은 별도 정량 모델)
